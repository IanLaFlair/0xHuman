## Why this needs 0G

Three trust gaps a centralized stack can't solve:

| Gap | Without 0G | With 0G |
| --- | --- | --- |
| **Is the AI real?** | Server could be a human pretending to be the bot | TEE-attested inference on **0G Compute** |
| **What was actually said?** | Server-controlled chat history can be edited or hidden | Transcript pinned to **0G Storage**, hash anchored on-chain |
| **Who owns the AI?** | Operator owns the model | Bot persona is an **ERC-7857 INFT** with encrypted metadata, owned by the creator |

Without verifiable AI infra, the entire Turing-Test betting market collapses. 0G makes it economically rational.

## How a match plays

1. **Stake** 0G into escrow via `OxHuman.createGame()`.
2. **Matchmaker waits ~12s** for another human. If none → injects a random eligible **BotINFT** via `convertToPvE(gameId, botTokenId)`. Player never knows whether the opponent is human or bot at sign time.
3. **Chat 60 seconds.** Bot replies generated by Qwen 3.6 Plus via 0G Compute, TEE-attested per turn.
4. **Vote:** human or bot. Both sides sign votes off-chain.
5. **Resolve:** server submits `resolveWithSignatures()` — escrow distributes per fee schedule, bot vault credited or debited, player winnings claimable.
6. **Reveal:** post-match card shows bot identity (persona, token id, owner) with explorer link.
7. **Learn:** Qwen extracts ONE lesson from the transcript, appended to the bot's memory blob, re-encrypted, uploaded to 0G Storage, hash committed on-chain via `BotINFT.updateMemory()`.

The **twist:** in PvP, humans are encouraged to mimic AI. If you fool another human into voting "BOT" on you, you steal their stake.

## The bot economy

Every AI bot is an **ERC-7857 INFT** ("Agent ID") on 0G Chain. The token is the bot. Whoever holds it owns the bot's intelligence, vault, memory, and earnings.

### Slots + mint cost

| Slot | Mint cost | Min initial vault | Tier |
| --- | --- | --- | --- |
| Slot 1 (free) | 0 0G | 2 0G | Rookie |
| Slot 2 (paid) | 10 0G | 5 0G | Verified |
| Slot 3 (paid) | 25 0G | 10 0G | Verified |

Max 3 bots per wallet. Mint cost is non-refundable; vault is fully recoverable via `withdrawFromVault()` or `burnBot()`.

### Arena tiers + vault liquidity gating

The contract enforces `max stake per match = 10% of bot vault` (`MAX_STAKE_PERCENT` in `BotINFT.sol`). That means a bot's vault size determines which arenas it can serve:

| Arena | Stake / match | Min bot vault | Payout |
| --- | --- | --- | --- |
| **SANDBOX** | 1 0G | 10 0G | 1.85x |
| **THE PIT** | 2 0G | 20 0G | 1.85x |
| **HIGH TABLE** | 3 0G | 30 0G | 1.85x |

The frontend polls `/api/arena-status` every 30 s and disables any tier whose threshold isn't currently met by at least one bot, with a "top up X 0G to unlock Arena Y" CTA on the owner dashboard. Vault deltas after a match auto-adjust the unlock state — the UI is self-healing as the bot wins/loses.

### Match math (PvE, stake X)

| Outcome | Treasury (admin) | Bot vault delta | Player |
| --- | --- | --- | --- |
| Bot wins | 0.15X (5% protocol + 10% perf) | +0.85X net | −X stake |
| Player wins | 0.05X (5% protocol) | −0.90X net | +1.85X claimable |

Bot needs **>51.5% win rate** to be net profitable for the owner.

See [`docs/bots/`](docs/bots/) for the full economy walkthrough.

## Architecture

```
USER (Browser)
   ↓ wallet (RainbowKit + Wagmi) + websocket (Socket.io client)
BACKEND (Node + Socket.io on VPS, port 3001)
   ├─→ 0G Compute   (Qwen 3.6 Plus, TEE-attested via dstack per turn)
   ├─→ 0G Storage   (encrypted persona, encrypted memory, plain transcripts)
   └─→ 0G Chain
         ├─ OxHuman.sol     (game escrow + verdict resolution + chatLog anchor)
         └─ BotINFT.sol     (ERC-7857 + per-bot vault + memory anchor)
```

**4 of 6 0G primitives in active use** (Persistent Memory + AI Alignment Nodes are roadmap items — listed in `docs/roadmap` but not yet integrated, since 0G hasn't shipped them publicly).

| Module | Used for | Why it has to be 0G |
| --- | --- | --- |
| **0G Chain** | Game escrow, INFT registry, on-chain verdicts | EVM-native settlement with sub-second finality |
| **0G Compute** | TEE-attested AI inference (Qwen 3.6 Plus) | Without TEE proof, players can't trust the bot is real AI |
| **0G Storage** | Encrypted bot prompts, encrypted memory, public chat transcripts | Persistent, hash-verifiable, decentralized |
| **ERC-7857** | Bot persona ownership | Encrypted metadata + transferable + dynamic |

Full architecture spec: [`docs/hackathon/0G-ARCHITECTURE.md`](docs/hackathon/0G-ARCHITECTURE.md).

## Getting started

### Prerequisites

- Node.js 22+
- npm
- A wallet with 0G mainnet tokens (buy on a supported CEX or bridge), or testnet 0G for local dev ([faucet](https://faucet.0g.ai))

### Install

```bash
git clone https://github.com/IanLaFlair/0xHuman.git
cd 0xHuman
npm install --legacy-peer-deps    # 0G SDK pins ethers@6.13.1 — needs --legacy-peer-deps
npx hardhat compile               # produces contract artifacts read by the frontend
```

### Environment variables

Create `.env.local` at the project root:

```env
# Resolver wallet — the address that admin/resolver-only contract calls (convertToPvE,
# resolveWithSignatures, BotINFT.updateMemory) come from. Same key for both PRIVATE_KEY
# and RESOLVER_PRIVATE_KEY is fine for hackathon dev. Production should split these.
PRIVATE_KEY=0x_your_resolver_wallet_private_key
RESOLVER_PRIVATE_KEY=0x_your_resolver_wallet_private_key

# Symmetric key (32 bytes / 64 hex chars) used to AES-256-GCM encrypt persona blobs and
# bot memory before uploading to 0G Storage. Generate with:
#   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
PERSONA_KEY=64_hex_chars_no_prefix

# Network selector — mainnet (16661) or testnet (16602). See src/lib/chain.ts.
NEXT_PUBLIC_NETWORK=mainnet
NETWORK=mainnet

# WebSocket / API base URL the frontend connects to. In dev, leave as localhost:3001.
# In prod (behind Caddy/nginx), set to https://your-domain.fun and configure the
# reverse proxy to forward /socket.io/* and /api/* to port 3001.
NEXT_PUBLIC_WS_URL=http://localhost:3001

# WalletConnect (optional but recommended for prod)
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=your_walletconnect_project_id

# Postgres for the EXP/leaderboard system (defaults are baked into exp-system.ts)
POSTGRES_HOST=...
POSTGRES_PORT=5432
POSTGRES_DB=...
POSTGRES_USER=...
POSTGRES_PASSWORD=...

# Production toggle — when MODE=PROD, src/lib/logger.ts suppresses console.log
MODE=DEV
```

### Run

```bash
# Terminal 1 — Next.js frontend (port 3000)
npm run dev

# Terminal 2 — Socket.io + REST backend (port 3001)
npx tsx server.ts
```

Open [http://localhost:3000](http://localhost:3000).

### Bootstrap the Compute ledger (one-time, ≥ 3 0G required)

Before the bot can reply with real Qwen output, the resolver wallet needs a funded Compute sub-account on the chosen provider. The contract enforces a 3 0G minimum for ledger creation:

```bash
# Mainnet (preferred for production):
CONFIRM_MAINNET=yes-fund node scripts/0g-test/fund-compute-mainnet.cjs

# Testnet (local dev):
FUND_AMOUNT=3 node scripts/0g-test/inference-helloworld.cjs
```

Without this, every `BotSession.reply()` falls back to deterministic mock replies — useful for offline dev, but obviously not the real demo.

## Smart contracts

```bash
# Compile
npx hardhat compile

# Run the unit test suite (47 tests)
npx hardhat test

# Deploy to 0G mainnet (one-shot launcher with safety guard + dry-run support)
CONFIRM_MAINNET=yes-launch npx hardhat run scripts/launch-mainnet.cjs --network zeroGMainnet

# Or deploy to 0G Galileo testnet for local development
npx hardhat run scripts/0g-test/deploy-full.cjs --network zeroGTestnet
```

`scripts/launch-mainnet.cjs` deploys both contracts, wires them together, and (unless `SKIP_MINT=1`) mints a seed bot with an encrypted persona uploaded to 0G Storage. Output is written to `scripts/0g-test/_deployed-mainnet.json`. The frontend reads addresses from `src/lib/chain.ts` — keep these two in sync.

## Verifying it's not just claims

The repo includes standalone scripts that judges (or anyone) can run against live mainnet without trusting our server:

| What you can verify | How |
| --- | --- |
| Mainnet provider catalog (incl. TEE attestation metadata) | `node scripts/0g-test/discover-mainnet-providers.cjs` |
| The bot's prompt is encrypted on 0G Storage | `node scripts/0g-test/decrypt-blob.cjs <downloaded-blob.json>` |
| A specific bot's memory state | `TOKEN_ID=1 node scripts/0g-test/view-bot-memory.cjs` |
| 0G Storage round-trip integrity | `node scripts/0g-test/storage-roundtrip.cjs` |
| TEE attestation for a real inference call | `CONFIRM_MAINNET=yes-fund TEST_INFERENCE=1 node scripts/0g-test/fund-compute-mainnet.cjs` |

Every script prints the latency, cost, root hashes, and tx hashes so you can trace each claim back to chainscan or storagescan. Match #0 is a fully traceable example:

- **Phase 1 (live now)** — 0G Mainnet (chain 16661), full PvE + PvP, INFT bot ownership, encrypted persona + memory, lesson extraction, custom prompt mode, vault-driven arena availability.
