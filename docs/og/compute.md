# 0G Compute (TEE Inference)

Every bot reply in 0xHuman is generated by **0G Compute Network** — a decentralized GPU marketplace where each provider runs models inside a Trusted Execution Environment and signs every response.

Mainnet has a larger and growing catalog. The bot architecture supports swapping models without code changes — `broker.inference.getServiceMetadata(providerAddress)` returns whatever the provider hosts.

---

## How a bot reply happens

1. **Server builds the system prompt**: persona body + memory RAG block + chat history.
2. **Server signs the request body** via `broker.inference.getRequestHeaders(providerAddress, body)`. The signature pins the exact bytes that will be sent.
3. **Provider validates the signature** and runs Qwen in its TEE. The provider also charges the caller's pre-funded sub-account (settled in batches).
4. **Provider returns the response + a chatID** in the `ZG-Res-Key` header.
5. **Server verifies the TEE attestation** via `broker.inference.processResponse(providerAddress, chatID)`. Confirms the response came from a real TEE, not a man-in-the-middle.
6. **Reply forwards to the player** with `verified: true` metadata.

## Cost reality

Per match:
- ~150 tokens of input (system prompt + history)
- ~50 tokens output per turn × 6–8 turns
- Total: < 0.0001 0G per match

The bottleneck is the **3 0G ledger minimum** to bootstrap a sub-account — a one-time deposit, not a per-call cost.

---

## Why this matters for the game

If players can't trust that the bot replies came from a real AI (and not a human pretending), the entire Turing Test betting market is defective. TEE attestation gives every reply a verifiable origin proof. Combined with the encrypted persona blob on 0G Storage, the operator can't substitute a different model or read the prompt — only the bot's owner can.
